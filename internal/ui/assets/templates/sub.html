订阅生成器

把筛选条件转成可直接复制的订阅和 API 链接。

这里适合做“出口前配置”：先挑格式，再选国家、协议、纯净度和延迟限制，最后一键复制完整链接。 生成结果默认会自动带上 token，适合客户端直接导入。

使用建议

  • Clash 用户优先选 `Clash YAML`，模板已默认带中国网络场景常用的 DNS、防污染和国内直连规则。
  • V2Ray 用户选 `V2Ray 订阅`，系统会输出 base64 链接。
  • 排查问题先切到 `JSON 节点列表` 看实际返回内容，再决定订阅参数。

生成参数

这些筛选条件会直接参与最终链接，可重复复制给不同客户端。

参数理解

为了让生成出来的链接更稳定，下面这些规则值得记一下。

`verify=1`

默认推荐开启。这样生成的订阅只会返回最近确认可用的节点，适合日常使用。

`limit`

订阅过大时客户端导入会变慢。一般 `50~300` 对多数客户端已经足够。

`countries`

如果你有特定地区需求，建议先按国家过滤，再叠加延迟和纯净度。

`protocols`

不同客户端支持的协议不同，不确定时先留默认，再在 JSON 模式检查结果。

`Clash YAML`

输出会附带自动选择、故障转移、地区分组，以及中国场景常用的 DNS 和直连分流规则。
