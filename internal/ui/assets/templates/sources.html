抓取源管理

优先保留高质量源，低收益源及时退避或停用。

系统会根据抓取成功率、有效产出和平均评分做源级排序。这个页面适合做两件事： 一是人工启停抓取源，二是临时补充新的订阅或文本源。

维护建议

  • 高失败率先看 `最近错误`，连续异常的源建议临时关闭。
  • 低收益如果 EMA 产出长期接近 0，可以减少它在 1c1g 上的资源占用。
  • 临时排查用“立即抓取”把 `next_fetch_at` 提前到现在，方便快速验证。

新增抓取源

支持 GitHub 原始文本、NodeMaven API、base64 订阅和 Clash YAML 订阅。

首次启动时系统会导入默认抓取源。这里更适合补充你自己的订阅地址，或者对已有源做启停管理。
