中文使用帮助

把出口 API、订阅格式和常用参数一次看明白。

这页面向真实使用场景编排：先讲鉴权，再讲接口，再给你可以直接复制改造的调用示例。 如果你主要通过浏览器使用后台，记得把 token 放到 URL；如果你走脚本或客户端，优先用请求头。

推荐用法

  • 浏览器后台访问 `/ui/...?...&token=你的token`。
  • 脚本和服务使用 `X-API-Key` 请求头，便于统一管理。
  • 订阅链接在“订阅生成”页先组合筛选条件，再复制完整 URL。

鉴权说明

两种方式都支持，按访问场景选就行。

X-API-Key: {{if .Token}}{{.Token}}{{else}}<你的token>{{end}}

或者

?token={{if .Token}}{{.Token}}{{else}}<你的token>{{end}}

说明：
1. 浏览器打开 UI 时，地址栏不会自动带 X-API-Key。
2. 所以 Web 后台、订阅链接更推荐用 ?token=...
3. curl、脚本、程序调用更推荐用 X-API-Key 请求头。

常用接口

健康检查和指标无需额外理解，出口接口重点看筛选参数。

  • 节点列表`GET /api/v1/nodes` 返回 JSON 数组。
  • 随机单节点`GET /api/v1/nodes/random`。
  • 纯文本订阅`GET /sub/plain`。
  • Clash 订阅`GET /sub/clash`。
  • V2Ray 订阅`GET /sub/v2ray`。
  • 监控接口`GET /metrics`、`/healthz`、`/readyz`。

通用筛选参数

下面这些参数基本所有出口都支持，可自由组合。

`countries`

国家白名单，逗号分隔，例如 `US,JP,SG`。

`protocols`

协议列表，例如 `http,socks5,ss,vmess,vless,trojan`。

`min_purity`

最低纯净度，范围 `0..100`，数值越高筛选越严格。

`max_latency_ms`

限制最大延迟，适合控制出口体验和可用性。

`fresh_within`

只返回最近多久检测过的节点，例如 `5m`、`30m`、`2h`。

`verify`

`1` 表示出口前只给有效节点；`0` 表示允许更宽松的候选池。

调用示例

把 `<BASE>` 替换为你的服务域名或 `http://127.0.0.1:38482` 即可。

# 1) 获取最近可用的 JSON 节点
curl -H 'X-API-Key: {{if .Token}}{{.Token}}{{else}}<你的token>{{end}}' \
  '<BASE>/api/v1/nodes?verify=1&limit=20&countries=JP,SG&max_latency_ms=800'

# 2) 取一个随机单节点
curl -H 'X-API-Key: {{if .Token}}{{.Token}}{{else}}<你的token>{{end}}' \
  '<BASE>/api/v1/nodes/random?verify=1&protocols=http,socks5'

# 3) 获取 Clash 订阅
curl -H 'X-API-Key: {{if .Token}}{{.Token}}{{else}}<你的token>{{end}}' \
  '<BASE>/sub/clash?verify=1&limit=200&min_purity=70'

# 4) 获取 V2Ray 订阅
curl -H 'X-API-Key: {{if .Token}}{{.Token}}{{else}}<你的token>{{end}}' \
  '<BASE>/sub/v2ray?verify=1&protocols=ss,vmess,vless,trojan'

常见问题

这些问题最常见，也最容易误判成系统故障。

  • 为什么浏览器打开 UI 提示未授权？因为浏览器地址栏不会自动带 `X-API-Key`，需要改成 `?token=...`。
  • 为什么 `verify=1` 时返回为空？说明当前筛选条件下 fresh 有效节点不足，可以放宽国家、纯净度或 fresh 时间窗。
  • 为什么节点很多但有效少？公共代理质量波动很大，建议在“抓取源”页停掉低收益源，减少无效检测开销。
